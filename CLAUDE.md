## What this project is

**formloop** lets an LLM agent generate a Typeform-quality form on-the-fly to ask a human structured questions, then resume execution when the form is submitted.

The thesis (developed during the night of 2026-05-09 with Jérôme):

1. Chat is a bad interface when an agent needs more than 2-3 structured inputs from a human.
2. `AskUserQuestion`-style harness primitives are too limited (4×4 max, no rich types).
3. Tally/Typeform aren't designed to be generated by code or to embed rich HTML context.
4. There's a real, growing pain point as agentic LLM usage spreads: how does an agent ask a human a complex question? How does the human stay in the loop without having to babysit the chat?

formloop is a **thin layer between any LLM agent and a human in the loop**, with a UI that goes beyond chat or markdown — embeddable HTML, clickable mockup grids, drag-and-drop, sliders with live previews.

## Origin story

This started as a discussion about Thariq Shihipar's article "The Unreasonable Effectiveness of HTML" (https://thariqs.github.io/html-effectiveness). Jérôme noticed that he often used Tally to communicate with Claude rather than typing in the terminal — a pattern that revealed a structural issue with chat-as-only-interface.

We then built (in one night):
1. A Vercel webhook bridge for Tally submissions
2. The custom Typeform-clone form engine (10 question types)
3. A Python skill (`ask-form`) that orchestrates the create → notify → poll → resume flow
4. This OSS-ready repo (where you are now)

After validating the concept end-to-end, Jérôme decided to turn it into an OSS project to (a) build dev credibility around AI tooling, (b) take a wedge in an emerging space before Anthropic/OpenAI absorb it natively.

## What works today (2026-05-09)

- **`apps/server/`** — fully working Next.js app. Deployed on Vercel under `manibors-projects/tally-bridge` at `https://tally-bridge-manibors-projects.vercel.app`. Includes:
  - `POST /api/forms` — create a custom form (auth: shared secret)
  - `GET  /forms/[id]` — render the form (Typeform-clone UI)
  - `POST /api/forms/[id]/submit` — receive submissions
  - `GET  /api/response/[id]` — poll for response (auth: shared secret)
  - `POST /api/webhook` — legacy Tally webhook receiver (still works)
  - Storage: Upstash Redis (Vercel Marketplace integration), TTL 1h on responses, 24h on specs.
  - 10 question types, all tested end-to-end via `apps/server/test-e2e.sh`.

- **`packages/sdk-python/ask_form.py`** — working CLI:
  - `create-custom --spec '{...}'` → creates a custom form on the formloop server, returns `{form_id, form_url}`
  - `create --spec '{...}'` → legacy Tally form mode (still functional, kept for now)
  - `wait --form-id <id> [--timeout 1800]` → polls until submitted, returns answers
  - `cleanup --form-id <id>` → deletes the form (Tally only)

- **Demo URL** (live for ~24h after creation, valid until 2026-05-10):
  `https://tally-bridge-manibors-projects.vercel.app/forms/PG3cmELG6d`

## Architecture (current)

```
LLM agent (Claude Code, GPT, etc.)
  │
  ├── 1. python ask_form.py create-custom --spec '{...}'
  │       → POST /api/forms (server)
  │       → server stores spec in Upstash Redis under spec:{id} (TTL 24h)
  │       → returns { form_id, form_url }
  │
  ├── 2. agent sends notification (email/Slack/Telegram) with form_url
  │
  ├── 3. Human opens form_url in browser
  │       → GET /forms/[id] (server-rendered Next.js page)
  │       → Loads Typeform-clone client component
  │       → Human fills form, clicks Submit
  │       → POST /api/forms/[id]/submit
  │       → Server stores response in Redis under response:{id} (TTL 1h)
  │
  └── 4. python ask_form.py wait --form-id <id>
          → polls GET /api/response/[id] every 5s
          → when ready, returns structured answers (label-keyed dict)
```

The legacy Tally path (form created on tally.so, webhook from Tally → server) writes to the same `response:{id}` key, so the polling logic is unchanged.

## Key design decisions

1. **Redis as ephemeral store, not a DB.** Responses TTL 1h, specs TTL 24h. No history, no persistence. Privacy-by-design + simplicity. Adding history is opt-in.

2. **Same `response:{id}` key for Tally and custom forms.** The polling endpoint serves both modes transparently. Migration path: agents can move from Tally mode to custom mode without changing their polling logic.

3. **HTML sanitization on the server.** Any HTML embedded by the agent (`html` blocks, `html-pick` options, `scale-preview` templates) goes through DOMPurify. No `<script>`, no event handlers, no iframes. SVG is allowed (charts, illustrations).

4. **Typeform-style UX deliberately copied.** One question per screen, vertical-slide transitions, large generous typography, accent-color (#6366f1) for selection. Familiar to anyone who's filled a Typeform.

5. **Keyboard shortcuts as first-class.** `a/b/c` to pick options, digits for scales, `o/n` for yes/no, arrows to navigate, Enter to advance. Power users feel at home.

6. **Mobile-first.** All views are responsive, drag-and-drop uses TouchSensor, scale-preview slider uses native range input.

## Conventions

- **Commits**: descriptive messages, "what + why", co-author credit when Claude Code is involved.
- **TypeScript**: `strict: true`, no `any`, prefer discriminated unions for question types.
- **Components**: server components by default; mark client components explicitly with `"use client"`.
- **Styling**: prefer CSS variables (in `globals.css`) and inline styles for one-off layout. Avoid Tailwind utility soup; the design tokens are in CSS.
- **Imports**: absolute paths via `@/` alias (configured in `tsconfig.json`).

## How to run locally (today)

```bash
# Server
cd apps/server
pnpm install   # or npm install
# Need ENV: WEBHOOK_SECRET, KV_REST_API_URL, KV_REST_API_TOKEN
# Currently: pull from Vercel with `vercel env pull` if linked
pnpm dev       # http://localhost:3000

# Python SDK
cd packages/sdk-python
python ask_form.py --help
# Note: BRIDGE_URL and WEBHOOK_SECRET hardcoded inside ask_form.py
# (this is one of the things to fix for OSS — see ROADMAP)
```
