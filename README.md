# formloop

> **Generate rich UI forms on-the-fly from LLM agents.** Let your agent ask humans complex questions visually ‚Äî with mockups, sliders, drag-and-drop, multi-choice ‚Äî instead of via chat.

When an LLM agent needs more than 2-3 inputs from a human, the chat interface breaks down: answers come in the wrong order, things get forgotten, the agent has to re-prompt. `formloop` solves this by letting the agent generate a Typeform-quality form on-the-fly, send the link, and resume with structured answers when the human submits.

It's a thin layer between any LLM agent and a human-in-the-loop, with a UI that goes far beyond what chat or markdown can do.

## What's in the box

10 question types, all generated from a single JSON spec:

**Simple** ‚Äî `mc`, `multi`, `text`, `textarea`, `yn`, `number`, `scale`
**Rich** ‚Äî `html-pick` (clickable mockup grid), `rank` (drag-and-drop ordering), `scale-preview` (slider with live HTML preview)
**Context** ‚Äî `html` (arbitrary HTML/SVG between questions for mockups, illustrations, instructions)

## Quick demo

```bash
# Start the server (Next.js, deployable on Vercel)
cd apps/server && pnpm install && pnpm dev

# In another terminal, drive it from Python
python packages/sdk-python/ask_form.py create-custom --spec '{
  "title": "Quick example",
  "blocks": [
    {"kind": "html-pick", "id": "design", "title": "Pick a hero",
     "options": [
       {"id": "a", "label": "Minimal",  "html": "<div>...</div>"},
       {"id": "b", "label": "Bold",     "html": "<div>...</div>"}
     ]},
    {"kind": "rank", "id": "order", "title": "Order these",
     "options": ["A", "B", "C"]}
  ]
}'
# ‚Üí returns { form_id, form_url }

# Open form_url, fill it, then:
python packages/sdk-python/ask_form.py wait --form-id <ID>
# ‚Üí returns { answers: { "Pick a hero": "b", "Order these": ["B", "A", "C"] } }
```
